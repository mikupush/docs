# Installation

In this guide, you will learn how to install the MikuPush server using Docker Compose, a Debian package, or a generic tarball for any Linux distribution.

:::tip Build it yourself
If you prefer to compile the server from source instead of using pre-built artifacts, check the [Building from Source](./building-from-source.mdx) guide.
:::

## Prerequisites

Before starting, ensure you have a **PostgreSQL 17** database available. If you use the Docker Compose method, a database container is included in the example.

## Install with Docker Compose (Recommended)

Using Docker is the easiest way to get MikuPush running. We recommend using Docker Compose to manage the server and its database.

### 1. Create a `docker-compose.yml`

Create a new directory for MikuPush and create a `docker-compose.yml` file:

```yaml
services:
  server:
    image: mikupush/server:latest
    ports:
      - "8080:8080"
    volumes:
      - ./data:/srv/data
      - ./config.yaml:/srv/config.yaml:ro # optional, only if you want to configure the server though the configuration file
    environment:
      - MIKU_PUSH_DATABASE_HOST=postgres
      - MIKU_PUSH_DATABASE_PORT=5432
      - MIKU_PUSH_DATABASE_NAME=miku_push
      - MIKU_PUSH_DATABASE_USER=miku_push
      - MIKU_PUSH_DATABASE_PASSWORD=miku_push

  postgres:
    image: postgres:17
    volumes:
      - postgres_data:/var/lib/postgresql/data
    environment:
      - POSTGRES_DB=miku_push
      - POSTGRES_USER=miku_push
      - POSTGRES_PASSWORD=miku_push

volumes:
  postgres_data:
```

### 2. Configuration

You can download a template for the `config.yaml` file:

```bash
curl -L https://raw.githubusercontent.com/mikupush/server/refs/heads/main/config.example.yaml -o config.yaml
```

If you need to customize the server configuration, you can edit this file or use Environment Variables. 

Check the [Configuration](./configuration.mdx) page for a full list of available options and their priorities.

**Example (Docker Environment Variables):**
You can also add configurations directly to your `docker-compose.yml`:
```yaml
    environment:
      - MIKU_PUSH_LOG_LEVEL=debug
      - MIKU_PUSH_UPLOAD_MAX_SIZE=100000000
```

### 3. Start the server

Run the following command to start the containers in the background:

```bash
docker compose up -d
```

The server will be available at `http://localhost:8080`. You can confirm it is running by following the [Verify](#6-verify) steps.

## Native Installation (Debian/Ubuntu)

If you prefer a native installation, we provide `.deb` packages for `amd64` and `arm64` architectures.

### 1. Download and Install

You can always download the latest version of the package directly from GitHub:

<LatestReleaseLinks />

### 2. Configuration

The configuration file is located at `/etc/io.mikupush.server/config.yaml`. Edit it to configure your database connection or other settings:

```bash
sudo nano /etc/io.mikupush.server/config.yaml
```

If you need to know more about the available options (logging, upload limits, etc.), please check the [Configuration](./configuration.mdx) page.

### 3. Service Management

The package automatically installs and enables a systemd service. You can manage it with the following commands:

```bash
# Start the service
sudo systemctl start mikupush-server

# Check the status
sudo systemctl status mikupush-server

# Restart after configuration changes
sudo systemctl restart mikupush-server
```

To confirm the server is responding, see the [Verify](#6-verify) steps.

## Generic Linux Installation (tar.gz)

For any Linux distribution with **systemd**, you can install MikuPush from the compressed tarball available on the [GitHub releases page](https://github.com/mikupush/server/releases).

The archive contains the following files:

| File | Description |
| :--- | :--- |
| `mikupush-server` | Server binary. |
| `config.yaml` | Default configuration file. |
| `dist/` | Pre-built web UI static files and templates. |
| `mikupush-server.service` | systemd unit file. |

### 1. Download and Extract

Create the installation directory and extract the archive:

```bash
sudo mkdir -p /opt/mikupush-server
```

<LatestReleaseTarLinks />

### 2. Create a Dedicated User

Create a system user to run the service:

```bash
sudo useradd -r -s /usr/sbin/nologin mikupush
```

### 3. Create Data and Log Directories

```bash
sudo mkdir -p /var/lib/io.mikupush.server/data
sudo mkdir -p /var/log/io.mikupush.server

sudo chown -R mikupush:mikupush /var/lib/io.mikupush.server
sudo chown -R mikupush:mikupush /var/log/io.mikupush.server
sudo chown -R mikupush:mikupush /opt/mikupush-server
```

### 4. Configuration

Edit the included `config.yaml` to point to the correct paths and configure your database connection:

```bash
sudo nano /opt/mikupush-server/config.yaml
```

Since the binary, templates and static files live under `/opt/mikupush-server`, update the paths accordingly:

```yaml
server:
  static_directory: /opt/mikupush-server/dist/assets
  templates_directory: /opt/mikupush-server/dist

log:
  directory: /var/log/io.mikupush.server

upload:
  directory: /var/lib/io.mikupush.server/data
```

For all available options, see the [Configuration](./configuration.mdx) page.

### 5. Install the systemd Service

Copy the included unit file and update `ExecStart` to point to the binary and config file:

```bash
sudo cp /opt/mikupush-server/mikupush-server.service /etc/systemd/system/mikupush-server.service
```

Edit the unit file to match the installation path:

```bash
sudo nano /etc/systemd/system/mikupush-server.service
```

```ini
[Unit]
Description=The Miku Push! server
After=network-online.target
Wants=network-online.target

[Service]
ExecStart=/opt/mikupush-server/mikupush-server --config /opt/mikupush-server/config.yaml
Restart=on-failure
User=mikupush
Group=mikupush

[Install]
WantedBy=multi-user.target
```

Then enable and start the service:

```bash
sudo systemctl daemon-reload
sudo systemctl enable mikupush-server
sudo systemctl start mikupush-server
```

### 6. Verify

Check that the service is running:

```bash
sudo systemctl status mikupush-server
```

Then confirm the server is responding:

```bash
curl -i -H 'Accept: application/json' http://localhost:8080/health
```

You should see a response like:

```
HTTP/1.1 200 OK
content-type: application/json
content-length: 15

{"status":"up"}
```
