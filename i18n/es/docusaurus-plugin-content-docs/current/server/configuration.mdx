# Configuración

El servidor *Miku Push!* es altamente configurable. Puedes usar un archivo YAML (`config.yaml`), variables de entorno o confiar en los valores predeterminados incorporados.

## Prioridad de configuración

Cuando el servidor se inicia, resuelve los valores de configuración en el siguiente orden de precedencia (de mayor a menor):

1.  **Variables de Entorno**: Si una variable está establecida en el entorno, siempre tendrá precedencia sobre cualquier otra fuente.
2.  **Archivo YAML (`config.yaml`)**: Si no se encuentra ninguna variable de entorno, el servidor busca el valor en el archivo de configuración.
3.  **Valores predeterminados incorporados**: Si una configuración falta tanto en las variables de entorno como en el archivo YAML, el servidor utiliza su valor predeterminado interno.

:::info Ejemplo de Precedencia
Si defines `port: 8080` en `config.yaml` pero también estableces la variable de entorno `MIKU_PUSH_SERVER_PORT=9000`, el servidor escuchará en el puerto **9000**.
:::

## Archivo de configuración

Puedes especificar la ruta de un archivo de configuración explícitamente con la opción `-c` / `--config`:

```bash
mikupush-server --config /path/to/config.yaml
```

Si no se proporciona ninguna la opción `-c`, el servidor busca un archivo `config.yaml` en las siguientes ubicaciones (en orden) y usa el **primero que encuentre**:

<Tabs groupId="config-file-platform">
  <TabItem value="linux" label="Linux" default>

1. `config.yaml` (directorio de trabajo)
2. `~/.io.mikupush.server/config.yaml`
3. `~/.config/io.mikupush.server/config.yaml`
4. `/etc/io.mikupush.server/config.yaml`

  </TabItem>
  <TabItem value="macos" label="macOS">

1. `config.yaml` (directorio de trabajo)
2. `~/.io.mikupush.server/config.yaml`
3. `~/.config/io.mikupush.server/config.yaml`
4. `~/Library/Application Support/io.mikupush.server/config.yaml`

  </TabItem>
  <TabItem value="windows" label="Windows">

1. `config.yaml` (directorio de trabajo)
2. `%LOCALAPPDATA%\io.mikupush.server\config.yaml`

  </TabItem>
</Tabs>

Si no se encuentra ningún archivo en ninguna de estas ubicaciones, el servidor se inicia usando solo variables de entorno y valores predeterminados incorporados.

## Ajustes del servidor

Ajustes generales para el servidor de la API HTTP.

<Tabs groupId="config-format">
  <TabItem value="yaml" label="YAML" default>

```yaml
server:
  name: mikupush.io
  # icon: examples/server_icon.svg
  host: 0.0.0.0
  port: 8080
  static_directory: dist/assets
  static_base_path: /assets
  templates_directory: dist
```

| Clave | Predeterminado | Requerido | Descripción |
| :--- | :--- | :---: | :--- |
| `server.name` | `mikupush.io` | No | Nombre público del servidor usado en endpoints de metadatos (por ejemplo `/api/info`). |
| `server.icon` | Sin definir | No | Ruta opcional al archivo del icono del servidor usado por `/api/icon`. Formatos soportados: `.jpeg`, `.png`, `.gif`, `.webp`, `.svg`. |
| `server.host` | `0.0.0.0` | No | Dirección IP o nombre de host donde el servidor escuchará las conexiones. |
| `server.port` | `8080` | No | Puerto TCP del servidor. |
| `server.static_directory` | `dist/assets` | No | Directorio con activos estáticos. Puede ser una ruta relativa (resuelta desde el directorio donde se encuentra el ejecutable) o absoluta. |
| `server.static_base_path` | `/assets` | No | Ruta base de la URL para servir archivos estáticos. |
| `server.templates_directory` | `dist` | No | Directorio con plantillas HTML utilizadas por la interfaz de usuario web. Puede ser una ruta relativa (resuelta desde el directorio donde se encuentra el ejecutable) o absoluta. |

  </TabItem>
  <TabItem value="env" label="Variables de Entorno">

```bash
MIKU_PUSH_SERVER_NAME=mikupush.io
# MIKU_PUSH_SERVER_ICON=examples/server_icon.svg
MIKU_PUSH_SERVER_HOST=0.0.0.0
MIKU_PUSH_SERVER_PORT=8080
MIKU_PUSH_SERVER_STATIC_DIR=dist/assets
MIKU_PUSH_SERVER_STATIC_BASE_PATH=/assets
MIKU_PUSH_SERVER_TEMPLATES_DIR=dist
```

| Variable | Predeterminado | Requerido | Descripción |
| :--- | :--- | :---: | :--- |
| `MIKU_PUSH_SERVER_NAME` | `mikupush.io` | No | Nombre público del servidor usado en endpoints de metadatos (por ejemplo `/api/info`). |
| `MIKU_PUSH_SERVER_ICON` | Sin definir | No | Ruta opcional al archivo del icono del servidor usado por `/api/icon`. Formatos soportados: `.jpeg`, `.png`, `.gif`, `.webp`, `.svg`. |
| `MIKU_PUSH_SERVER_HOST` | `0.0.0.0` | No | Dirección IP o nombre de host donde el servidor escuchará las conexiones. |
| `MIKU_PUSH_SERVER_PORT` | `8080` | No | Puerto TCP del servidor. |
| `MIKU_PUSH_SERVER_STATIC_DIR` | `dist/assets` | No | Directorio con activos estáticos. Puede ser una ruta relativa (resuelta desde el directorio donde se encuentra el ejecutable) o absoluta. |
| `MIKU_PUSH_SERVER_STATIC_BASE_PATH` | `/assets` | No | Ruta base de la URL para servir archivos estáticos. |
| `MIKU_PUSH_SERVER_TEMPLATES_DIR` | `dist` | No | Directorio con plantillas HTML utilizadas por la interfaz de usuario web. Puede ser una ruta relativa (resuelta desde el directorio donde se encuentra el ejecutable) o absoluta. |

  </TabItem>

## Subida de archivos

Configura cómo el servidor maneja las subidas de archivos y la expiración.

<Tabs groupId="config-format">
  <TabItem value="yaml" label="YAML" default>

```yaml
upload:
  # Tamaño máximo en bytes (p.ej. 5GB)
  max_size: 5000000000
  # Directorio de almacenamiento
  directory: data
  # Expiración en segundos (p.ej. 7 días)
  expires_in_seconds: 604800
  # Intervalo de limpieza en segundos (p.ej. 1 hora)
  expiration_cleanup_interval_seconds: 3600
```

| Clave | Predeterminado | Requerido | Descripción |
| :--- | :--- | :---: | :--- |
| `upload.max_size` | Sin límite | No | Tamaño máximo de subida permitido en bytes. Si no se establece, no hay límite. |
| `upload.directory` | `data` | No | Directorio donde se almacenan los archivos subidos. Puede ser una ruta relativa (resuelta desde el directorio donde se encuentra el ejecutable) o absoluta. |
| `upload.expires_in_seconds` | Sin expiración | No | Tiempo de expiración para los archivos subidos en segundos. Si no se establece, los archivos no expiran. |
| `upload.expiration_cleanup_interval_seconds` | `3600` | No | Intervalo en segundos para comprobar y eliminar archivos expirados. |

  </TabItem>
  <TabItem value="env" label="Variables de Entorno">

```bash
MIKU_PUSH_UPLOAD_MAX_SIZE=5000000000 # 5GB
MIKU_PUSH_UPLOAD_DIRECTORY=data
MIKU_PUSH_UPLOAD_EXPIRES_IN_SECONDS=604800
MIKU_PUSH_UPLOAD_EXPIRATION_CLEANUP_INTERVAL=3600
```

| Variable | Predeterminado | Requerido | Descripción |
| :--- | :--- | :---: | :--- |
| `MIKU_PUSH_UPLOAD_MAX_SIZE` | Sin límite | No | Tamaño máximo de subida permitido en bytes. Si no se establece, no hay límite. |
| `MIKU_PUSH_UPLOAD_DIRECTORY` | `data` | No | Directorio donde se almacenan los archivos subidos. Puede ser una ruta relativa (resuelta desde el directorio donde se encuentra el ejecutable) o absoluta. |
| `MIKU_PUSH_UPLOAD_EXPIRES_IN_SECONDS` | Sin expiración | No | Tiempo de expiración para los archivos subidos en segundos. Si no se establece, los archivos no expiran. |
| `MIKU_PUSH_UPLOAD_EXPIRATION_CLEANUP_INTERVAL` | `3600` | No | Intervalo en segundos para comprobar y eliminar archivos expirados. |

  </TabItem>
</Tabs>

## Logging

Controla la verbosidad y la salida de los logs del servidor.

<Tabs groupId="config-format">
  <TabItem value="yaml" label="YAML" default>

```yaml
log:
  level: info
  file_prefix: server
  directory: /var/log/io.mikupush.server
  json: false
```

| Clave | Predeterminado | Requerido | Descripción |
| :--- | :--- | :---: | :--- |
| `log.level` | `info` | No | Nivel de registro. Valores posibles: `trace`, `debug`, `info`, `warn`, `error`. |
| `log.file_prefix` | `server` | No | Prefijo del nombre del archivo de logs. Por ejemplo, `server` produce archivos como `server.2025-12-11-14.log`. |
| `log.directory` | Específico de la plataforma | No | Directorio donde se almacenarán los logs. Predeterminado a `/var/log/io.mikupush.server` en Linux, `~/Library/Logs/io.mikupush.server` en macOS, y `%LOCALAPPDATA%\io.mikupush.server\logs` en Windows. |
| `log.json` | `false` | No | Si es `true`, la salida del log usa formato JSON. |

  </TabItem>
  <TabItem value="env" label="Variables de Entorno">

```bash
MIKU_PUSH_LOG_LEVEL=info
MIKU_PUSH_LOG_FILE_PREFIX=server
MIKU_PUSH_LOG_DIRECTORY=/var/log/io.mikupush.server
MIKU_PUSH_LOG_JSON=false
```

| Variable | Predeterminado | Requerido | Descripción |
| :--- | :--- | :---: | :--- |
| `MIKU_PUSH_LOG_LEVEL` | `info` | No | Nivel de registro. Valores posibles: `trace`, `debug`, `info`, `warn`, `error`. |
| `MIKU_PUSH_LOG_FILE_PREFIX` | `server` | No | Prefijo del nombre del archivo de logs. Por ejemplo, `server` produce archivos como `server.2025-12-11-14.log`. |
| `MIKU_PUSH_LOG_DIRECTORY` | Específico de la plataforma | No | Directorio donde se almacenarán los logs. Predeterminado a `/var/log/io.mikupush.server` en Linux, `~/Library/Logs/io.mikupush.server` en macOS, y `%LOCALAPPDATA%\io.mikupush.server\logs` en Windows. |
| `MIKU_PUSH_LOG_JSON` | `false` | No | Si es `true`, la salida del log usa formato JSON. |

  </TabItem>
</Tabs>

## Modo de depuración

Estos ajustes están destinados al **desarrollo local** de la interfaz de usuario web.

Cuando el modo de depuración está habilitado, el servidor obtiene plantillas HTML del dev server de [Astro](https://astro.build/) que esta en ejecución en lugar de leerlas del sistema de ficheros local. Esto te permite trabajar en la interfaz de usuario web con *Hot Module Reloading (HMR)* mientras el servidor maneja las solicitudes de la API e inyecta los metadatos necesarios en las plantillas antes de servirlas al navegador.

Cuando el modo de depuración está deshabilitado (el valor predeterminado), las plantillas se leen del directorio especificado por `server.templates_directory`.

<Tabs groupId="config-format">
  <TabItem value="yaml" label="YAML" default>

```yaml
debug:
  enable: false
  astro_dev_server: http://localhost:4321/
```

| Clave | Predeterminado | Requerido | Descripción |
| :--- | :--- | :---: | :--- |
| `debug.enable` | `false` | No | Habilita el modo de depuración. Cuando está habilitado, el servidor obtiene plantillas del dev server de Astro en lugar de archivos locales. |
| `debug.astro_dev_server` | `http://localhost:4321/` | No | URL del dev server de Astro. Solo se usa cuando `debug.enable` es `true`. |

  </TabItem>
  <TabItem value="env" label="Variables de Entorno">

```bash
MIKU_PUSH_DEBUG_ENABLE=false
MIKU_PUSH_DEBUG_ASTRO_DEV_SERVER=http://localhost:4321/
```

| Variable | Predeterminado | Requerido | Descripción |
| :--- | :--- | :---: | :--- |
| `MIKU_PUSH_DEBUG_ENABLE` | `false` | No | Habilita el modo de depuración. Cuando está habilitado, el servidor obtiene plantillas del dev server de Astro en lugar de archivos locales. |
| `MIKU_PUSH_DEBUG_ASTRO_DEV_SERVER` | `http://localhost:4321/` | No | URL del dev server de Astro. Solo se usa cuando `debug.enable` es `true`. |

  </TabItem>
</Tabs>
