# Compilando desde la Fuente

Esta guía explica cómo compilar el servidor Miku Push! desde la fuente en Linux y producir un binario, un paquete `.deb` o un archivo `.tar.gz`.

## Prerrequisitos

Necesitas tener instaladas las siguientes herramientas:

| Herramienta | Versión | Propósito |
| :--- | :--- | :--- |
| [Rust](https://www.rust-lang.org/tools/install) | 1.88+ | Compila el binario del servidor. |
| [Node.js](https://nodejs.org/) | 22+ | Construye la interfaz de usuario web (Astro). |
| npm | (incluido con Node.js) | Instala las dependencias de JavaScript. |
| git | | Clona el repositorio. |
| dpkg | (preinstalado en Debian/Ubuntu) | Solo se requiere para construir el paquete `.deb`. |

## Clonar el Repositorio

```bash
git clone https://github.com/mikupush/server.git
cd server
```

## Instalar Dependencias de JavaScript

```bash
npm install
```

## Construir

El proyecto usa un `Makefile` que orquesta tanto la construcción de la interfaz de usuario web como la del binario de Rust. Hay dos perfiles de construcción disponibles: **debug** y **release**.

### Solo Binario

Para compilar solo el binario del servidor:

```bash
# Construcción de depuración (compilación más rápida, incluye símbolos de depuración)
cargo build

# Construcción de lanzamiento (optimizada)
cargo build --release
```

El binario resultante estará en:

| Perfil | Ruta |
| :--- | :--- |
| Debug | `target/debug/mikupush-server` |
| Release | `target/release/mikupush-server` |

:::info
El binario por sí solo no es suficiente para ejecutar el servidor con la interfaz de usuario web. También necesitas construir la interfaz de usuario web (ver a continuación) para que el servidor pueda servir los archivos estáticos y las plantillas.
:::

### Interfaz de Usuario Web

Para construir la interfaz de usuario web (Astro):

```bash
npm run build
```

Esto genera el directorio `dist/` que contiene los activos estáticos y las plantillas HTML.

### Construcción Completa (Binario + Interfaz de Usuario Web + Paquetes)

El `Makefile` proporciona objetivos que construyen todo y producen paquetes distribuibles:

```bash
# Paquetes de depuración
make linux-debug

# Paquetes de lanzamiento (recomendado)
make linux-release
```

Estos objetivos se ejecutan, en orden:
1. `npm run build` — construye la interfaz de usuario web en `dist/`.
2. `cargo build` — compila el binario del servidor.
3. `scripts/deb-package.sh` — crea el paquete `.deb`.
4. `scripts/tar-package.sh` — crea el archivo `.tar.gz`.

## Artefactos de Salida

Después de una construcción completa, los archivos generados se encuentran en `target/<profile>/bundle/`:

| Artefacto | Ruta |
| :--- | :--- |
| paquete `.deb` | `target/<profile>/bundle/mikupush-server_<version>_<arch>.deb` |
| archivo `.tar.gz` | `target/<profile>/bundle/mikupush-server_<version>_<arch>.tar.gz` |
| Binario del servidor | `target/<profile>/mikupush-server` |

Donde `<profile>` es `debug` o `release`, `<version>` se lee del archivo `VERSION`, y `<arch>` es la arquitectura de tu sistema (p.ej. `amd64`, `arm64`).

### Qué hay Dentro de Cada Paquete

El **paquete `.deb`** instala los archivos en las rutas estándar del sistema:

| Archivo | Instalado en |
| :--- | :--- |
| Binario del servidor | `/usr/bin/mikupush-server` |
| Configuración | `/etc/io.mikupush.server/config.yaml` |
| Interfaz de usuario web | `/usr/share/io.mikupush.server/dist/` |
| Servicio systemd | `/etc/systemd/system/mikupush-server.service` |

El **archivo `.tar.gz`** contiene todos los archivos en una estructura plana, listos para ser extraídos en cualquier directorio:

| Archivo | Descripción |
| :--- | :--- |
| `mikupush-server` | Binario del servidor. |
| `config.yaml` | Archivo de configuración por defecto. |
| `dist/` | Archivos estáticos y plantillas de la interfaz de usuario web precompilados. |
| `mikupush-server.service` | Archivo de unidad de systemd. |

Para instrucciones de instalación usando estos artefactos, consulta la página de [Instalación](./installation.mdx).

## Imagen de Docker

También puedes construir la imagen de Docker localmente. El `Dockerfile` usa una construcción de múltiples etapas: una etapa compila el binario de Rust, otra construye la interfaz de usuario web con Node.js, y la etapa final produce una imagen mínima basada en `debian:bookworm-slim`.

Desde la raíz del repositorio:

```bash
docker build -t mikupush/server:local .
```

Luego ejecútala:

```bash
docker run -d 
  --name mikupush-server 
  -p 8080:8080 
  -v "./data:/srv/data" 
  mikupush/server:local
```

Para una configuración completa de Docker Compose con PostgreSQL, consulta la página de [Instalación](./installation.mdx#install-with-docker-compose-recommended).
